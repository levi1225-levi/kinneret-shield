    │

    │   Sine wave tone generation for audio feedback.
    │   5 sound effects:
    │   - Boot chime (ascending C5→G5)
    │   - Card beep (high C6)
    │   - Success tone (ascending C5→G5)
    │   - Failure tone (descending buzz)
    │   - Emergency alarm (siren 1kHz↔500Hz)
    │   Volume control (0-255).

    │   Device authentication via API key in headers.
    │   JSON parsing with ArduinoJson.

    ├── storage_handler.cpp (390 lines)

    │   - Device ID, Room name, API key
    │   Local attendance logging:
    │   - CSV format with timestamp and card UID
    │   Offline sync queue:
    │   - Queues card taps when server unreachable
    │   - Auto-syncs when reconnected
    │   - Max 500 events in queue
    │   File operations with proper error handling.
    │
    ├── ota_handler.h (60 lines)

    │   Auto-restart on completion.

KNOWN LIMITATIONS
=================

1. WiFi only supports 2.4 GHz (most ESP32s lack 5 GHz)
2. NFC reader has ~4-7 cm read range (design limitation)
3. OLED display is monochrome (no color graphics)
4. Audio output requires external speaker (no onboard speaker)
5. OTA updates require >2 MB free flash space
6. WebSocket requires HTTPS for production use (implement in network_handler)
7. No battery backup (requires external UPS for power stability)
8. Max 500 offline events before queue overflow

SUPPORT & TROUBLESHOOTING
=========================

Common Issues:
1. NFC not detected → Check I2C address, power, connections
2. WiFi won't connect → Reset to AP mode via 192.168.4.1
3. Server unreachable → Check firewall, API key, network
4. Display not showing → Verify I2C address, power, contrast
5. Audio quiet → Increase volume, check speaker connections
